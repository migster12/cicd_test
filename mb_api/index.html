mobile_banking

CyberONE Mobile Banking

A mobile banking app by CyberONE Online Solutions

Features

  • x Transfer Funds
  • x Manage your savings accounts
  • x View your statement of account
  • x Enroll other savings accounts
  • x View time deposit accounts
  • x View shared capital accounts
  • x Manage your mobile account and secure it with MPIN

Framework

Working properly using Flutter 3.7.1 with sound null safety. Support for Android and iOS.

Minimum Requirements

Android

- OS: Android 6.0 and above
- Memory: 1GB or more
- Storage: 80MB of internal storage or more
- Internet Connectivity

Note: Install on internal storage only (Do not move to SD card)

Warning: Turn off Developer mode on your device and make sure device is not rooted.

iOS

- OS: iOS 9.0 and above
- Memory: 1GB or more
- Storage: 80MB of internal storage or more
- Internet Connectivity

Warning: Make sure device is not jailbroken

Development

  1. After cloning, make sure to add the following lines to the local.properties file in the android level
    flutter.buildMode=debug
    flutter.versionName=2.0.0
    flutter.versionCode=116
    flutter.minSdkVersion=23
    flutter.targetSdkVersion=33
    flutter.compileSdkVersion=33
  1. Create a key.properties file in the android level
    storePassword=CoreONE
    keyPassword=CoreONE
    keyAlias=upload
    storeFile=upload-keystore.jks

  1. Upload your upload-keystore.jks file in the android/app level

  2. Replace the dependencies in External Libraries/Flutter Plugins/flutter_jailbreak_detection-1.8.0/android/build.gradle file from

    dependencies {
        implementation "org.jetbrains.kotlin:kotlin-stdlib-jdk7:$kotlin_version"
        implementation 'com.github.scottyab:rootbeer:0.1.0'
    }

to

    dependencies {
        implementation "org.jetbrains.kotlin:kotlin-stdlib-jdk7:$kotlin_version"
        implementation 'com.scottyab:rootbeer-lib:0.1.0'
    }

Project Hierarchy

|__ android
|__ ios
|__ doc |__ assets |__ lib |____ components |____ models
|____ screens |____ services

  1. android - Necessary files for building android version. It is already pre configured, nothing should change.
  2. ios - Necessary files for buiding ios version. It is already pre configured, nothing should change.
  3. doc - Source code documentation. Please check index.html
  4. assets - Contains fonts, images and json to be loaded in project.
  5. lib - Contains main logic of the app. Subfolders are divided based on usage.
  6. components - Resuable widgets that can be used in any part of the screens.
  7. models - Mostly structured data for server response values.
  8. screens - Widgets for user interaction with transactions.
  9. services - Services use to easily request to bank server for transactions, security feature implementations, utilities functions etc.
